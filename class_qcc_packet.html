Detailed Description

The QccPacket class is used for sending custom TCP-packets over the network.

The QccPacket class defines the different packet types QCC needs for the network communication beween the QCC-server and the QCC-client.

It is also used for sending QCC-packets over the network through the TCP.

Each QCC-packet consists of three parts:

  1. The size of the actual packet in bytes (quint32).
    The size is defined by the size of the packet-type plus the size of the payload.
  2. The type of the packet (encoded as qint32).
  3. The actual data of the packet (payload) (the data-type depends on the packet-type).
    The payload can also be empty, because some packet-types don't need any data.

ConnectionAccepted 

The connection has been accepted by the QCC-server.

This packet has no payload.

ConnectionRefused 

The connection has been refused by the QCC-server.

This packet has no payload.

UserRegister 

The QCC-client sends a user registration.

Payload:

  1. QString(username)
  2. QString(passwort) - SHA-256 encoded hash
  3. QByteArray(publicKey) - RSA public key in DER-format
RegisterSuccess 

The QCC-server reports that the user registration has been successful.

This packet has no payload.

RegisterFailure 

The QCC-server reports that an error occured with the user registration.

Payload:

  1. QString(reason) - reason of the failure

The QCC-client sends a user authentication (login).

Payload:

  1. QString(username)
  2. QString(passwort) - SHA-256 encoded hash
  3. QByteArray(publicKey) - RSA public key in DER-format
AuthenticationSuccess 

The QCC-server reports that the user authentication has been successful.

This packet has no payload.

AuthenticationFailure 

The QCC-server reports that an error occured with the user authentication.

Payload:

  1. QString(reason) - reason of the failure
RequestAuthorization 

The QCC-client sends an authorization request to add another user to his contacts.

Payload:

  1. QString(username)

The username gets replaced by the server with the username of the sender, so that the receiver knows from where the authorization request come from.

AuthorizationAccepted 

The authorization request has been accepted.

Payload:

  1. QString(username)
  2. qint32(status) - user status
  3. QByteArray(publicKey) - RSA public key in DER-format

The status and publicKey is appended by the server.

AuthorizationDeclined 

The authorization request has been declined.

Payload:

  1. QString(username)

The QCC-server reports that an error occured with the authorization request.

Payload:

  1. QString(reason) - reason of the failure
RequestContactList 

The QCC-client requests his list of contacts.

This packet has no payload.

ContactList 

The QCC-server sends a list of the user's contacts.

Payload:

  1. qint32(count) - user count
  2. list of contacts (count user objects)
    • user object
      1. QString(username)
      2. qint32(status) - user status
      3. QByteArray(publicKey) - RSA public key in DER-format
    • user object ...
ContactStatusChanged 

The QCC-Server sends a status-changed-packet.

The status-changed-packet is send to all users which have the user who changed the status on their contact list.

Payload:

  1. QString(username) - user who changed the status
  2. qint32(status) - user status
  3. QByteArray(publicKey) - RSA public key in DER-format
RemoveContact 

The QCC-client sends a request to remove a user from his contacts.

Payload:

  1. QString(username)
ContactRemoved 

The QCC-Server confirms that the user has been removed from the contacts.

Payload:

  1. QString(username)
Message 

RSA-encrypted message from one user to another.

Payload:

  1. qint32(messageId) - the message id is used for association with the following MessageSuccess packet (if any)
  2. QString(receiver|sender) - username of the sender/receiver
  3. QByteArray(message) - RSA-encrypted message
MessageSuccess 

Confirms that the message has been successfully delivered.

Payload:

  1. qint32(messageId) - the message id is used for association with the message
  2. QString(sender) - username of the sender
MessageFailure 

The QCC-server reports that an error occured with the message.

Payload:

  1. qint32(messageId) - the message id is used for association with the message
  2. QString(receiver) - username of the receiver
  3. QString(reason) - reason of the failure
IllegalPacketType 

Used for illegal packet types that don't match any valid packet type.
